1) Persistent Memory and Linux

Linux Foundation Collaboration Summit 2013, April 15-17, 2013 · Parc 55 Hotel (San Francisco, CA) にて、Ric Wheeler (Red Hat) が、LinuxでのNon-volatileメモリ対応に関するセッションを行ったようです。

Persistent Memory and Linux: New Storage Technologies and Interfaces Ric Wheeler, Senior Manager, Kernel File & Storage Team Red Hat, Inc. https://events.linuxfoundation.org/images/stories/slides/lfcs2013_wheeler.pdf

lwn.netでも、このセッションの記事がアップされているようですが、私はまだこの記事を読んでいません。 LFCS: Preparing Linux for nonvolatile memory devices

既存のSSDデバイスや今後のPCI-SSDに対するLinuxのチャレンジについて語っていて、 PCI-SSDデバイスで100万IOPS/deviceを実現するためには、Linuxカーネルのブロック層だけでなく ファイルシステムやさらにはデータベースアプリケーションを変更する必要があるようです。

アプリケーション開発者はゆっくりと新しいハードウェアの恩恵が得るため、 これからの数年は read/write といった従来のAPIを利用し続け、 一部のハイエンドアプリケーションが新しいAPIやプログラミングモデルを利用するようです。

Storage Network Industry Association (SNIA) のワーキンググループの一つである Non Volatile Memory Programming TWG にて、新しいプログラミングモデルについて既に議論がなされています。

こういうところで活躍できるような技術者になれるといいなあ。
